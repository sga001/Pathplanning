# RRT / RRT* Implementation Speedup Plan (issue #10)

**Goal:** Make `planners/rrt.py` and `planners/rrt_star.py` materially faster so the
periodic-replan variants (`rrt_replan`, `rrt_star_replan`) stop timing out under
~20-obstacle traffic — **without changing the computed path, cost, or the
byte-identical `trace.jsonl` guarantee**. Pure implementation work.

**Approach:** Profile-first. `cProfile` a representative hard seed to confirm the
bottleneck, then optimize only what the profile proves hot. The strong-model
consult predicts the dominant cost is the **collision-sampling inner loop**
(`segment_is_clear_grid` runs a ~21-sample Python loop per edge calling
`world_to_grid` per sample; RRT* calls it 40–80× per node across choose-parent +
rewire), **not** the O(N²) node-array rebuild (a vectorized-C call over a
few-hundred-node array — microseconds). The primary fix is therefore an
rrt.py-**local, allocation-free scalar line-of-sight (LOS) helper** that keeps
`segment_is_clear_grid`'s exact sample count and arithmetic but replaces the
numpy-scalar `np.floor`/`np.clip` calls with stdlib `math.floor`/`min`/`max`, so
the accept/reject bool is identical (verified by the TC47 equivalence fuzz). An incremental numpy node-array is
a low-risk secondary, included only if the profile shows the nearest-neighbor scan
is non-negligible. Full vectorized LOS is held in reserve, used only if the scalar
rewrite does not clear the timeouts and only behind a bool-equivalence fuzz gate.

## Scope

- **In scope:**
  - Edit `planners/rrt.py` and `planners/rrt_star.py` only.
  - Add a permanent equivalence test (`TC47`) to `arena/arena.py`'s `--check` suite
    that fuzzes the rrt-local LOS helper against `segment_is_clear_grid`.
  - Profile a representative hard seed; establish a representative-subset before/after
    baseline (timeouts + wallclock + byte-identical traces).
  - Update `CLAUDE.md` (planner-family note + check count 47 → 48) and per-file commits.
- **Out of scope:**
  - Editing `planners/_grid.py`, `manual_astar.py`, or `planners/d_star_lite.py`
    (D* Lite's DNFs were already fixed by the merged deferred-settle work; if the
    shared `segment_is_clear_grid` is the dominant cost it is addressed only via an
    rrt.py-LOCAL helper, never by editing the shared function).
  - Changing any algorithmic knob (`RRT_SEED`, `RRT_MAX_ITERS`, `RRT_STEP`,
    `RRT_GOAL_BIAS`, `RRT_GOAL_TOLERANCE`, `RRT_STAR_NEIGHBOR_RADIUS`) — frozen.
  - New dependencies (no scipy / cKDTree). Pure numpy + stdlib.
  - Integer-DDA line walk or spatial-hash nearest-neighbor — both change the
    accept/reject bool or argmin tie order → break byte-identical traces. Disqualified.
  - Forcing a genuinely-unsolvable seed; those are flagged for issue #9.
  - Full 50-seed confirmation across both families (verification is scoped to a
    representative subset; full-50 is an optional follow-up the user runs).

## Decisions

- **Profile-first, collision-LOS primary** — the consult vetoed the original
  "node-array first" priority. Profiling confirms the target before code changes; the
  expected primary is the scalar local LOS, with node-array secondary.
- **Allocation-free scalar LOS over full vectorization** — same arithmetic structure
  as `segment_is_clear_grid` makes the bool identical (strong empirical evidence via
  TC47, not a formal proof) with far less determinism-verification burden. Full
  vectorization stays in reserve behind the SAME TC47 fuzz gate (it reuses TC47; no new gate).
- **Local helper, not a shared edit** — `segment_is_clear_grid` lives in the frozen
  `_grid.py`; the speedup is a parallel copy inside `rrt.py`. Drift is guarded by a
  permanent equivalence TC (TC47), not just inline comments.
- **Knobs frozen, output byte-identical** — keeps the Mission scatter-plot comparison
  honest (the change measures the same search, only faster).
- **Representative-subset verification, bounded cost** — baseline uses a small canonical
  seed set with a reduced wall to find slow episodes cheaply; survivors after the fix get
  a big-wall re-run to classify slow vs unsolvable.
- **New branch `rrt-perf` off main, commit per file** — independent verify/revert per the
  issue's "one file per change."

- [ ] **AC3 (LOS equivalence):** The rrt-local LOS helper returns the **identical bool**
  as `segment_is_clear_grid` over ≥10⁵ random `(p0, p1)` segments on random occupancy

- [ ] **AC4 (speedup):** `wallclock_per_step` for `rrt_replan` and `rrt_star_replan` drops

- [ ] **AC5 (timeouts):** Every representative-subset seed that previously failed to complete

## The LOS-equivalence trap (load-bearing for AC1/AC3)

The local helper MUST reproduce `segment_is_clear_grid`'s exact bool. The enumerated
equivalence obligations (every one is load-bearing for AC1/AC3):

1. **Clip-then-read, never OOB-reject.** `segment_is_clear_grid` **clips then reads
   occupancy** and never rejects a sample purely for being out of bounds: `world_to_grid`
   clips an OOB point to an edge cell, and the sample is rejected only if that (in-bounds)
   edge cell is occupied. (The `is_cell_in_bounds` disjunct in `segment_is_clear_grid` is
   dead code after `world_to_grid` already clips — so the fast helper may read
   `grid_cells[clamped_row, clamped_col]` directly, but it must clamp identically.) The
   helper must **not** add an explicit OOB rejection, or the bool flips and the trace diverges.
2. **Length MUST be `math.sqrt(dx*dx + dy*dy)` — `math.hypot` is FORBIDDEN.** `math.sqrt(dx*dx+dy*dy)`
   is bit-identical to the original `np.linalg.norm(segment)` (0 mismatches / 1M random
   inputs); `math.hypot(dx, dy)` diverges in ~17% of inputs (170,799 / 1,000,000) because its
   extended-precision intermediate yields a different last bit. A single-ULP length difference
   propagates through the sample count and flips the bool. **Concrete counterexample:**
   `dx=1.0, dy=2.4000000000000004` → norm-path `sample_count=26` vs hypot-path `27`.
3. **Sample count `max(2, math.ceil(length / (resolution * 0.5)))`** — use `math.ceil`
   (equals `int(np.ceil(...))` for the finite positive lengths here), and the identical
   `ratio = i / count` progression.
4. **`math.floor((x - offset) / resolution)` clamped with `min/max` must equal
   `int(np.clip(np.floor(...), 0, n-1))`** for the finite in-range floats here (it does — same
   IEEE division then floor — confirmed over 2M random + edge inputs; TC47 guards it).
5. **The `length < 1e-9` degenerate branch** (`segment_is_clear_grid` lines 203-205) must be
   replicated: a near-coincident segment checks only the START cell (clip-then-read). RRT
   produces near-zero edges via `_steer` when a sample ≈ the nearest node, so this path is reachable.

**TC47 stratified coverage (mandatory — uniform-interior sampling alone would never hit
the load-bearing branches).** From a fixed RNG, apportion the ≥10⁵ segments across ALL of:
1. **Out-of-bounds endpoints** — one or both of p0/p1 with negative coords and/or coords past
   width/height (exercises the clip-to-edge-cell-then-read path, the named LOS-equivalence trap).
2. **Degenerate near-zero segments** — p0/p1 within sub-1e-9 of each other, plus exactly-equal
   endpoints (exercises the `length < 1e-9` branch).
3. **Length spread** — segment lengths spanning multiple `sample_count` regimes (short
   single-sample, mid, long multi-sample) so any length/sample-count divergence surfaces.
4. **In-bounds ordinary segments** — the baseline stratum.
The random grids MUST place occupied cells **adjacent to all four edges** (not only interior
obstacles) — an OOB endpoint clipped to an edge cell only distinguishes the two
implementations when that edge cell is sometimes occupied. Also assert the helper computes
`length` via `math.sqrt` (bit-identical to `np.linalg.norm`), explicitly catching a future
`math.hypot` swap.

**Test data:** Canonical seeds via `derive_episode_seeds(20260605, N)`; representative
subset = first ~5–8 derived seeds. TC47's RNG is fixed-seed so TC47 is itself deterministic.
**Run command:** `.venv\Scripts\Activate.ps1; python arena/arena.py arena/arena_v1.yaml --check`

## Notes for Implementer

- **Determinism is the gate, not the profiler.** Every code change is accepted only if
  the before/after `trace.jsonl` is byte-identical for all four keys AND TC47 passes.
  Capture the "before" hashes (T2) on the *unmodified* tree before any edit.
- **Exact-bool reproduction (T3) is the whole risk.** Re-read `segment_is_clear_grid` and
  `world_to_grid` side by side; replicate the clip-then-read semantics precisely. The fuzz
  TC (T5) is what makes editing around the frozen shared function safe — write T5 alongside
  T3 and run it continuously while iterating T3.
- **Node-array is optional and secondary.** Only add it if T1's profile shows the NN scan
  is a real fraction of runtime. Shipping it alone is explicitly *not* the win. If included,
  preallocate with doubling and slice `[:N]`; never change the float values or scan order.
- **Bounded baseline cost (T2).** Full-episode DNF episodes can each burn the per-episode
  wall (~600 s real). To find slow seeds cheaply, run the subset with a *reduced* wall and
  treat "did not complete + high `wallclock_per_step`" as the slow signal. Reserve a couple
  of full-episode confirmations for after the fix.
- **`_append_clear_waypoints` lives in `_grid.py`** (out of scope). The waypoint-bisection
  LOS calls there are NOT editable; only `rrt_plan`'s per-edge check and RRT*'s
  choose-parent/rewire checks (which call `segment_is_clear_grid` directly from the planner
  modules) are in scope. Confirm in T1 whether the waypoint path even shows up hot.
- **rrt_star imports from rrt.** `_segment_clear_fast` defined in `rrt.py` is imported by
  `rrt_star.py` (same pattern as the existing helper imports), so T4 depends on T3.
- **Legacy in-code AC labels.** `rrt.py` / `rrt_star.py` comments reference `AC4`, `AC5`,
  `AC7`, `AC7-obs` from the ORIGINAL Phase-6 RRT plan — those do NOT map to this spec's
  AC1–AC6. Don't renumber the code comments; just don't cross-reference them against this
  spec. (e.g. in-code "AC7-obs" = the RRT*-vs-RRT planned-cost observation = this spec's AC2.)
- **Post-LOS re-profile (T4).** T1 profiles the *unmodified* code, where the slow LOS masks
  the NN rebuild's share — so T1's NN fraction is a lower bound on its post-fix significance.
  Re-profile after T3/T4 land the LOS speedup before deciding the node-array isn't worth it.
- **Rollback:** per-file commits mean any single change that breaks AC1 is reverted in
  isolation without losing the others.
